# Agents.md — PDF Chat RAG WordPress Plugin

## Project Identity
A WordPress plugin enabling users to chat with uploaded PDF documents via a 
Retrieval-Augmented Generation (RAG) pipeline. The entire RAG pipeline runs 
in pure PHP — no Python microservice, Docker, or external server required.

## Tech Stack
- **Backend**: PHP 8.0+, WordPress 6.0+, Composer (PSR-4)
- **Frontend**: React via `@wordpress/scripts` (wp-scripts), `@wordpress/api-fetch`, `@wordpress/components`
- **Database**: WordPress `wpdb` with custom tables + JSON storage for embeddings
- **AI/Vector**: Google Gemini API via `wp_remote_post()`, PHP brute-force cosine similarity
- **PDF Parsing**: `smalot/pdfparser` via Composer
- **Patterns**: Singleton (Core\Plugin), Repository, Interface/Contract, REST API MVC

## Architecture Rules

### 1. PHP Namespace & Autoloading
- **Namespace Root**: `PDFChatRAG\`
- **PSR-4 Mapping**: `PDFChatRAG\` → `src/`
- All classes must be `declare(strict_types=1);`
- Only `Core\Plugin` is a Singleton. All other classes are instantiated normally.

### 2. Dependency Wiring
- `Plugin` creates `GeminiClient`, `PhpVectorStore`, and `ChatRepository`, then passes them to `Pipeline`
- `Plugin` passes `Pipeline` to `RestApi`
- `RestApi` instantiates controllers inline; `ChatController` no longer needs Pipeline DI (self-instantiates)
- `PdfController` self-instantiates `WpPdfParser`, `TextChunker`, `GeminiClient`, `PhpVectorStore`
- `SettingsController` is stateless, reads/writes WordPress options directly

### 3. WordPress REST API
- **Namespace**: `pdf-chat-rag/v1`
- **Controllers**: Located in `src/Api/Controllers/`
- **Permissions**:
  - Public endpoints (`/chat`, `/chat/history`) use `__return_true` but validate
    `session_id` and sanitize all inputs.
  - Admin endpoints (`/pdf/upload`, `/settings`) use `AuthMiddleware::checkAdminPermission`.
- **Nonce Handling**: Frontend uses localized `wp_create_nonce('wp_rest')` passed via
  `X-WP-Nonce`. `api-fetch` middleware is configured in `assets/src/utils/api.js`.

### 4. Database
- Use `$wpdb` with prepared statements only.
- Custom tables: `{prefix}pdf_chat_history`, `{prefix}pdf_index`, `{prefix}pdf_vectors`
- Migrations are idempotent (use `dbDelta` via `Activator` on plugin activation).
- Embeddings stored as JSON in `longtext` columns in `{prefix}pdf_vectors`.

### 5. Services & Contracts
- All external dependencies (LLM, Vector DB, PDF Parser) must implement an interface
  from `src/Services/Contracts/`.
- `GeminiClient` implements `LlmProviderInterface` — handles Gemini's OpenAI-compatible `/chat/completions` (`gemini-2.5-flash`) and native `embedContent` (`gemini-embedding-001`) API via `wp_remote_post()`.
- `PhpVectorStore` implements `VectorStoreInterface` — stores embeddings as JSON, brute-force cosine similarity in PHP.
- `WpPdfParser` implements `PdfParserInterface` — wraps `smalot/pdfparser`.
- `TextChunker` is a utility (no interface) — sentence-aware splitting with overlap.
- `Pipeline` orchestrates: Embed query → Vector search → Format history → LLM generate → Save.

### 6. Frontend (wp-scripts)
- **Entry Points**: `assets/src/admin/index.js`, `assets/src/frontend/index.js`, `assets/src/shortcode/index.js`
- **Build Output**: `build/` (generated by `npm run build`)
- **Webpack**: Custom `webpack.config.js` extends `@wordpress/scripts` with multi-entry
- **Asset Loading**: `Frontend\AssetLoader` enqueues scripts and localizes
  `window.pdfChatRag` with `restUrl` and `nonce`.
- **Shortcode**: `[pdf_chat]` renders inline React chat widget. Assets only loaded when shortcode is present.
- **Components**:
  - `ChatBox` — Shared chat UI component (used by floating widget and shortcode)
    - Message bubbles with distinct colors: user messages use purple gradient (`#6366F1`), assistant messages use white background
    - Timestamps on each message
    - Typing indicator with animated bouncing dots
    - Custom SVG send button with loading spinner
  - `ChatWidget` — Floating bubble + panel wrapper around `ChatBox`
    - Gradient purple header (`#6366F1` → `#8B5CF6`) with PDF icon
    - Rounded-square toggle button with gradient and glow shadow
    - Smooth fade-in/slide-up animations
    - Custom scrollbar styling
- Use WordPress components (`@wordpress/components`) for admin UI.
- **CSS Design System**:
  - Primary accent: Indigo gradient (`#6366F1` → `#8B5CF6`)
  - User bubbles: Gradient with white text, rounded corners (`border-radius: 16px 16px 4px 16px`)
  - Assistant bubbles: White with subtle shadow, rounded corners (`border-radius: 16px 16px 16px 4px`)
  - Background: Light gray (`#fafafa`) with subtle radial gradient accents
  - Animations: `pdf-chat-fade-in`, `pdf-chat-slide-up`, `pdf-chat-typing-bounce`, `pdf-chat-spin`

### 7. Security
- Sanitize: `sanitize_text_field()`, `sanitize_textarea_content()` for inputs.
- Capabilities: Always check `current_user_can()` for admin routes (use `AuthMiddleware`).
- File Uploads: Restrict to `.pdf`, verify MIME type via `wp_check_filetype`, use `wp_handle_upload()`.
- No raw SQL concatenation. Always `$wpdb->prepare()`.
- API Key: `PDF_CHAT_RAG_GEMINI_API_KEY` constant in `wp-config.php` overrides DB-stored key and is never exposed via REST API.

### 8. Shortcode
- `[pdf_chat]` — Renders inline chat widget
- `[pdf_chat session_id="custom"]` — Uses a specific session ID
- Registered in `Frontend\AssetLoader`
- Entry point: `assets/src/shortcode/index.js`
- Reuses `ChatBox` component from `assets/src/frontend/components/ChatBox.js`

## AI/Vector Stack (PHP-Native)

The plugin uses pure PHP for the entire RAG pipeline. No external microservice is required.

- **PDF Parsing**: `smalot/pdfparser` (Composer) extracts text from text-based PDFs.
- **Text Chunking**: `Services\TextChunker` splits text into ~1,000 character chunks with 200-char overlap.
- **Embeddings**: `Services\GeminiClient` calls Gemini's `gemini-embedding-001` API via `wp_remote_post()`.
- **Vector Storage**: `Services\PhpVectorStore` stores embeddings as JSON in `{prefix}pdf_vectors`
  and performs brute-force cosine similarity in PHP. Suitable for <20,000 chunks.
- **LLM Generation**: `Services\GeminiClient` calls Gemini's `gemini-2.5-flash` model via OpenAI-compatible endpoint.
- **Pipeline**: `Services\Rag\Pipeline` orchestrates: Embed → Retrieve → Generate → Store.

### Scaling Path
If vector search performance degrades, implement a new `VectorStoreInterface` (e.g., `PineconeStore`)
that calls a hosted vector DB via HTTP. The rest of the plugin remains unchanged.

## File Organization
When adding features:
- New API endpoints → `src/Api/RestApi.php` + `src/Api/Controllers/`
- New middleware → `src/Api/Middleware/`
- New database tables → `src/Database/Migrations/` + update `Activator`
- New data access → `src/Database/Repository/`
- New external service → Implement interface in `src/Services/Contracts/` first
- New admin UI → `assets/src/admin/components/`
- New frontend component → `assets/src/frontend/components/`
- New shortcode UI → `assets/src/shortcode/`

## REST API Schema
### POST `/wp-json/pdf-chat-rag/v1/chat`
**Request:**
```json
{
  "message": "string (required)",
  "session_id": "string (optional, generated if missing)"
}
```

**Response:**
```json
{
  "success": true,
  "response": "string",
  "session_id": "string",
  "context": []
}
```

### GET `/wp-json/pdf-chat-rag/v1/chat/history?session_id=xxx`
Returns array of history objects ordered chronologically (reversed, limit 20).

### POST `/wp-json/pdf-chat-rag/v1/pdf/upload`
Multipart form data. Requires `manage_options`.
**Response:** `{ "success": true, "document_id": "uuid", "filename": "doc.pdf", "chunks": 42 }`

### GET `/wp-json/pdf-chat-rag/v1/settings`
Returns: `{ "gemini_api_key": "AIza...****" }`. Requires `manage_options`.

### POST `/wp-json/pdf-chat-rag/v1/settings`
Saves: `{ "gemini_api_key": "..." }`. Requires `manage_options`.
